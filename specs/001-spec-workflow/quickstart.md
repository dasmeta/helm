# Quickstart: dasmeta/helm and Spec Workflow

**Feature**: 001-spec-workflow  
**Date**: 2026-03-09

## Using the charts (consumer)

```bash
# Add the Dasmeta Helm repo
helm repo add dasmeta https://dasmeta.github.io/helm
helm repo update

# List available charts (see ./charts for source)
helm search repo dasmeta/

# Install a chart (example: base)
helm upgrade --install my-release dasmeta/base -f my-values.yaml

# Or with inline overrides
helm upgrade --install my-release dasmeta/base --set key=value
```

## Developing / adding a chart (contributor)

1. **Branch**: Create a feature branch `001-<feature-name>` (or use existing).
2. **Spec**: Run `/speckit.specify` with a short description to create or edit `specs/<branch>/spec.md`.
3. **Plan**: Run `/speckit.plan` to generate/update plan, research, data-model, quickstart, contracts.
4. **Tasks**: Run `/speckit.tasks` to generate `specs/<branch>/tasks.md`.
5. **Implement**: Follow tasks; add or modify charts under `charts/<chart-name>/`.
6. **Validate** (per constitution):
   ```bash
   helm lint charts/<chart-name>
   helm template test charts/<chart-name>
   ```
7. **Pre-commit** (optional): `git config --global core.hooksPath ./githooks`

## Spec workflow artifacts (this feature)

- **spec.md**: User stories and requirements.
- **plan.md**: Technical context and Constitution Check.
- **research.md**: Decisions and rationale.
- **data-model.md**: Chart/values structure (no DB).
- **contracts/**: Chart consumption and values contract.
- **tasks.md**: Generated by `/speckit.tasks` (Phase 2).
